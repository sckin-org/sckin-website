# sckin-website

Source for [sckin.org](https://sckin.org) — the public website of **SCKIN**
(Sickle Cell Knowledge and Information Network), a 501(c)(3) nonprofit making
information about sickle cell disease universally accessible.

## Stack

- **Next.js** (App Router, TypeScript)
- **Tailwind CSS v4**, driven by CSS custom properties — see
  [`docs/tokens.md`](docs/tokens.md)
- **Content:** Markdown + YAML frontmatter in [`content/`](content/)
- **Internationalization:** a `[locale]` route segment via
  [`src/middleware.ts`](src/middleware.ts); English-only at launch with
  **unprefixed** URLs (`/mission`, not `/en/mission`). See
  [`src/lib/i18n.ts`](src/lib/i18n.ts).
- **Hosting:** **Vercel (Pro)** — auto-deploys on push to `main`, preview URLs
  per PR.

> **Design status:** the visual design ships separately as a Claude Design
> handoff. Components are currently unstyled placeholders. See
> [`docs/design-spec.md`](docs/design-spec.md).

## Local development

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # production build
```

Requires **Node 20 or newer**.

### Environment variables

Copy [`.env.example`](.env.example) to `.env.local` and fill in
`GOOGLE_SHEETS_WEBHOOK_URL` (the Google Apps Script Web App that receives the
Contact and SickleCellPedia Pro form submissions — see
[`docs/apps-script-endpoint.js`](docs/apps-script-endpoint.js)). It is read
server-side only and must never be committed. On Vercel, set it under
Project → Settings → Environment Variables.

## Content workflow

**All page copy lives in Markdown, not in code.** To change what the site says,
edit a file under `content/` and push — Vercel rebuilds and redeploys `main`
automatically.

```
edit content/*.md  →  git push origin main  →  Vercel rebuilds  →  live
```

### How content maps to pages

| File                              | Page / route           |
| --------------------------------- | ---------------------- |
| `content/home.md`                 | `/`                    |
| `content/mission.md`              | `/mission`             |
| `content/about.md`                | `/about`               |
| `content/impact.md`               | `/impact`              |
| `content/sicklecellpedia.md`      | `/sicklecellpedia`     |
| `content/sicklecellpedia-pro.md`  | `/sicklecellpedia-pro` |
| `content/responsible-ai.md`       | `/responsible-ai`      |
| `content/publications.md`         | `/impact/publications` |
| `content/contact.md`              | `/contact`             |
| `content/news/*.md`               | `/news` (one file per post) |

> URLs are unprefixed for English. When a second locale is added it is prefixed
> (e.g. `/fr/mission`, `/fr/impact/publications`) while English URLs stay the
> same. See the Internationalization note above.

Each file has **structured fields in YAML frontmatter** (the part between the
`---` fences) and **prose in the body** below. Keep the frontmatter field names
as they are — they are typed in [`src/lib/content.ts`](src/lib/content.ts) and
consumed by the page components. Placeholder values are marked `[PLACEHOLDER]`.

Routes `/donate`, `/whatsapp`, and `/feedback` are not content-driven — their
copy lives in the page components under `src/app/`.

### News content contract (for the ingestion DAG)

News posts under `content/news/*.md` are **generated by an external DAG** that
ingests articles, classifies them, and writes one Markdown file per post. **The
website only displays them — it does not own the taxonomy.** Whoever builds the
DAG must emit exactly this frontmatter:

```markdown
---
title: "Headline"              # string, required
date: "2026-06-01"             # YYYY-MM-DD, required; controls sort (newest first)
summary: "One-sentence summary." # string, required
source_url: "https://…"        # string; link to the original coverage
topics: ["Newborn screening", "Policy"]   # array of strings, may be empty
geographies: ["Nigeria", "Global South"]  # array of strings, may be empty
image: "my-post.jpg"           # optional; lives in public/images/
---

Body prose (optional).
```

How the site treats this contract:

- **Filter options are derived dynamically at build time** from the distinct
  `topics` and `geographies` values across all posts (see `getNewsFacets()` in
  [`src/lib/content.ts`](src/lib/content.ts)). There is **no hardcoded topic or
  geography list** anywhere in the site — add a post with a new value and it
  appears as a filter option automatically.
- Filters are client-side and **combinable** (topic **AND** geography), with an
  "All …" reset per facet.
- Posts are sorted by `date` descending.
- Edge cases are handled gracefully: empty `topics`/`geographies` arrays, a
  missing `source_url` or `image`, and an **empty `content/news/`** (the page
  renders with no posts rather than crashing).

> **Value normalization is the DAG's responsibility, not the site's.** The site
> shows the distinct values verbatim. If the DAG emits both `"Nigeria"` and
> `"nigeria"`, the site will show **two** separate filter options. Normalize
> casing/spelling upstream before writing the Markdown.

There is a `content/news/example-post.md` you can copy or delete.

## Project layout

```
content/            Markdown page + news content (edit this to change copy)
docs/               requirements, design spec, design tokens, Apps Script endpoint
public/images/      images referenced from content
src/app/[locale]/   routes (App Router), scoped to the locale segment
src/app/api/         form submission route handlers (never localized)
src/middleware.ts   locale routing: rewrites unprefixed → default locale
src/components/     shared components (header, footer, news browser, form)
src/lib/content.ts  typed Markdown/frontmatter loader (locale-aware)
src/lib/i18n.ts     locale config + href helper
src/lib/nav.ts      navigation model
amplify.yml         legacy — retained from the earlier hosting evaluation; unused now that hosting is Vercel
```

## Deployment

Pushing to `main` triggers a **Vercel** production deploy; every PR gets a
preview URL. The domain cutover to `sckin.org` (currently on GoDaddy) happens
after launch. (`amplify.yml` is a leftover from the earlier hosting evaluation
and can be removed.)
